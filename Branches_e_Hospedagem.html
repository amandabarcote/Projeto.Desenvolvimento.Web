O que é uma Branche

Uma branche é uma extensão do seu projeto inicial. Ela pode ser uma nova funcionalidade, atualização, correção de erro ou qualquer coisa que precise ser desenvolvida separadamente do projeto principal, pois o site não pode ficar em constante mudança (um site no ar não pode sofrer atualizações constantes). Um site que não precise disso não precisa de uma branch, pois não faz mal algum sofrer atualizações frequentes. Um bom exemplo de implementação de branches é no caso de um desenvolvimento em equipe, onde cada um desenvolve uma parte do site separadamente, para que, assim, ao final, o código seja mesclado com a branch master ou main. Todas as branches são feitas a partir da master.


Como fazer uma branche

Primeiro, deve-se criar um repositório. Depois, desenvolve-se a branch main ou master (isso deve ser feito porque a branch posterior é apenas uma adição à branch principal).

Passo a Passo no GitHub desktop para fazer uma Branche: Current branche → New branche → Coloca o nome da branche → Create branche → Publish branche.


Hospedagem

Para conseguir hospedar um site no GitHub Pages, é necessário que seu site esteja publico e não seja comercial.


Como hospedar um site

Selecione o repositório → Settings → Role até GitHub Pages e clique em "Check it out here!" → Vá em Source e mude de None para Main → Salve as configurações clicando em Save. Você receberá um link com seu site já hospedado. Veja-o no GitHub Pages.

Observações: Você deve deixar o index.html na raiz do repositório, sem colocá-lo em nenhuma pasta. Se ele for colocado em alguma pasta, o sistema não conseguirá encontrá-lo e apenas hospedará o README.
