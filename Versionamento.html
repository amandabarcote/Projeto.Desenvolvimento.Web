Software de Versionamento

Para que serve o Git?
O Git serve para organizar as versões de um software, organizar as versões, versionamento e melhora a organização da equipe, uma vez que o software se responsabiliza pela organização.


Controle de versões


Sobre os tipos de Versionamentos (VCS)
Antes de falar dos tipos, tenho que explicar o básico sobre como ocorre o vercionamento, basicamente, ele é uma versão de seu código, que pode ser guardado em sua própria máquina ou mandado para um servidor central/remoto. O Git oferece o vercionamento tanto local quando para um servidor remoto. Na imagem acima vemos ilustrado a conexão entre um repositório local e um repositório remoto (o repositorio, falando em termos brutos, é uma pasta com seus arquivos).

O vercionamento centralizado faz o versionamento num servidor central, é como se todos os repositórios de um escritório fossem ligados um servidor próprio do escritório, para ter sucesso é necessario estar sempre conectado ao servidor central, esse sistema é mais antigo porem ainda está em uso, os VCS que representam esse sistema é o CVS e SVN. Ele não tem versionamento local e é totalmente dependente do servidor.

O versionamento distribuido tem um repositório local e um repositório remoto, o usuário tem o controle se quer atualizar o repositoro remoto ou se quer deixar o repositório apenas em sua maquina, este sistema é mais novo e melhor, os softwares mais notaveis são o BitKeep e o Git.

Git x GitHub


Qual a diferença entre Git e GitHub?
Em poucas palavras, a diferença é que o Git faz o versionamento e o GitHub hospeda esse versionamento. Os dois são distintos, mas se complementam. Juntos, eles se fundem e formam o melhor software de versionamento (na minha opinião). O Git serve para versionar qualquer código-fonte, e com o GitHub é possível acessar seu código de qualquer lugar. O GitHub também pode ser considerado uma rede social.
